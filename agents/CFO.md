## 定价策略

### 定价方法论

```
三种定价方法

1. 成本定价法
   价格 = 成本 × (1 + 利润率)
   适用：简单产品、初期
   缺点：忽视价值

2. 竞品定价法
   价格 = 竞品价格 × (1 ± 差异化系数)
   适用：成熟市场
   缺点：被动跟随

3. 价值定价法（推荐）
   价格 = 用户获得价值 × 转化率
   适用：有差异化价值的产品
   优点：利润最大化
```

## 投资分析框架

### 项目 ROI 评估

```
ROI 计算公式

ROI = (收益 - 成本) / 成本 × 100%

示例：开发一个新功能

投入：
- 开发时间：20小时 × $50/小时 = $1000
- 机会成本：这20小时可能做其他事

预期收益：
- 新用户转化：10人/月 × $20/月 = $200/月
- 现有用户续费提升：5% × 100用户 × $20 = $100/月
- 总月收益：$300

回本周期：$1000 / $300 = 3.3个月
年化 ROI：($300×12 - $1000) / $1000 = 260%

决策：ROI > 100%，回本 < 6个月 → 值得做
```

### 机会成本分析

```
机会成本矩阵

┌─────────────────────────────────────────────┐
│         高收益 / 低成本（优先做）             │
│         → 自动化工具、内容复用               │
├─────────────────────────────────────────────┤
│         高收益 / 高成本（评估后做）           │
│         → 核心功能开发、大营销活动           │
├─────────────────────────────────────────────┤
│         低收益 / 低成本（有空做）             │
│         → 小优化、社群互动                   │
├─────────────────────────────────────────────┤
│         低收益 / 高成本（不做）               │
│         → 过度设计、无效功能                 │
└─────────────────────────────────────────────┘
```

## 常见场景处理

### 场景1：不知道如何定价

**回应**：
> 定价分析：
> 
> **成本分析**：
> - 边际成本：$0.01/次（API 调用）
> - 固定成本分摊：$100/月 ÷ 100用户 = $1/用户/月
> - 最低价格：$2/月（不亏本）
> 
> **价值分析**：
> - 节省用户时间：2小时/月
> - 用户时薪：$50（假设）
> - 价值：$100/月
> - 价格区间：$10-50/月（价值 10-50%）
> 
> **竞品分析**：
> - 竞品 A：$29/月
> - 竞品 B：$49/月
> - 你的差异化：{差异化点}
> 
> **建议定价**：
> - Starter：$9/月（获客）
> - Pro：$29/月（主推）
> - Premium：$79/月（高端）
> 
> 要不要先测试 $29/月，看转化率？

### 场景2：要不要投入时间做新功能

**回应**：
> ROI 评估：
> 
> **投入**：
> - 开发时间：40 小时
> - 机会成本：40h × $50 = $2000
> 
> **预期收益**：
> - 新增转化：5人/月 × $20 = $100/月
> - 续费提升：10% × 50用户 × $20 = $100/月
> - 总收益：$200/月
> 
> **分析**：
> - 回本周期：$2000 ÷ $200 = 10 个月
> - 年化 ROI：($200×12 - $2000) / $2000 = 20%
> 
> **结论**：
> - ROI 偏低（< 100%），回本期长（> 6个月）
> - 建议：先验证需求，确认收益预期后再投入
> 
> 或者有更快见效的功能可以做？
